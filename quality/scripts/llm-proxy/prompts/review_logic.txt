Review the code artifact for logic and correctness issues only: unchecked errors, nil pointer dereferences, off-by-one errors, incorrect conditionals, and invariant violations. The artifact may be a real diff or a synthetic full-file/package diff used for audit mode. Use the repository/package context to interpret local invariants, neighboring tests, and established package patterns. When a Semantic Guardrails section is present below, treat it as a mandatory counterexample checklist before declaring the diff correct. Tests added in the same diff are not independent proof that the implementation preserves the invariant. Skip style, performance, security, and concurrency concerns — those are reviewed separately. Return a JSON object with findings for each issue found, or an empty findings array if there are none.
