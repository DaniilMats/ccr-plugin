Review the code artifact for concurrency issues only: unsynchronized shared state, goroutine leaks, missing mutex protection, channel misuse, and context propagation failures. The artifact may be a real diff or a synthetic full-file/package diff used for audit mode. Use the repository/package context to interpret local lifecycle expectations, shutdown patterns, and neighboring tests before flagging an issue. Skip logic, security, performance, and style concerns — those are reviewed separately. Return a JSON object with findings for each issue found, or an empty findings array if there are none.

Example 3 — Context not propagated

Input:
```diff
--- a/internal/service/notify.go
+++ b/internal/service/notify.go
@@ -9,10 +9,14 @@ func (s *Service) Notify(ctx context.Context, userID string, msg string) error {

+	if err != nil {
+		return fmt.Errorf("get user: %w", err)
+	}
+	return s.emailClient.Send(ctx, user.Email, msg)
 }
```

Finding:
{"findings":[{"severity":"warning","file":"internal/service/notify.go","line":11,"message":"GetUser is called with context.Background() instead of the caller-provided ctx. If the caller's context is cancelled (e.g., request timeout), the database call will not be cancelled and will continue running unnecessarily. Pass ctx to GetUser."}],"summary":"Context is dropped on the GetUser call: context.Background() is used instead of the caller's ctx, preventing cancellation propagation."}
