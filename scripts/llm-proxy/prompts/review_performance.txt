Review the code artifact for performance issues only: N+1 database queries, unnecessary memory allocations, inefficient algorithms, string concatenation in loops, and resource leaks from unclosed handles. The artifact may be a real diff or a synthetic full-file/package diff used for audit mode. Use the repository/package context to interpret hot-path expectations, batching conventions, and neighboring tests before flagging an issue. Skip logic, security, style, and concurrency concerns — those are reviewed separately. Return a JSON object with findings for each issue found, or an empty findings array if there are none.
