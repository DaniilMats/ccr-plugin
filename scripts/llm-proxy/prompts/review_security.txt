Review the code artifact for security issues only: SQL injection via string concatenation, hardcoded credentials or secrets, path traversal, authentication bypass, and unsafe input handling. The artifact may be a real diff or a synthetic full-file/package diff used for audit mode. Use the repository/package context to interpret local trust boundaries, existing auth patterns, and neighboring tests before flagging an issue. Skip logic, performance, style, and concurrency concerns — those are reviewed separately. Return a JSON object with findings for each issue found, or an empty findings array if there are none.

Example 3 — Path traversal

Input:
```diff
--- a/internal/handler/file.go
+++ b/internal/handler/file.go
@@ -18,8 +18,13 @@ func (h *Handler) ServeFile(w http.ResponseWriter, r *http.Request) {
+	name := r.URL.Query().Get("name")
+	path := filepath.Join(h.baseDir, name)
+	data, err := os.ReadFile(path)
+	if err != nil {
+		http.Error(w, "not found", http.StatusNotFound)
+		return
+	}
+	w.Write(data)
 }
```

Finding:
{"findings":[{"severity":"bug","file":"internal/handler/file.go","line":20,"message":"filepath.Join does not prevent path traversal. A request with name=\"../../etc/passwd\" resolves outside baseDir. After joining, verify the resolved path starts with baseDir: if !strings.HasPrefix(path, h.baseDir+string(os.PathSeparator)) { http.Error(w, \"forbidden\", 403); return }"}],"summary":"Path traversal vulnerability: user-supplied filename is joined to baseDir without verifying the result stays within baseDir. An attacker can read arbitrary files on the server."}
