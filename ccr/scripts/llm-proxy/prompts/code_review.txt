## Review Criteria

Evaluate the code across these dimensions:

### READABILITY
- Naming clarity: variables, functions, types, and packages follow Go naming conventions
- Code formatting: consistent with gofmt output
- Import grouping: stdlib, external, internal separated by blank lines
- Package structure: logical organization, no circular dependencies
- Comments: exported symbols documented, complex logic explained

### CORRECTNESS
- Error handling: all errors checked, wrapped with context using %w
- Nil safety: nil pointer dereferences guarded, zero values handled
- Race conditions: shared state properly synchronized, goroutines safe
- Edge cases: boundary conditions, empty inputs, large inputs considered
- Logic bugs: incorrect conditionals, off-by-one errors, wrong assumptions

### PERFORMANCE
- Unnecessary allocations: slices/maps pre-allocated where size is known
- N+1 queries: database calls inside loops
- Missing context propagation: context.Context not threaded through call chain
- Inefficient algorithms: O(n²) where O(n) is possible
- String concatenation in loops: use strings.Builder

### SECURITY
- Input validation: user inputs validated and sanitized
- SQL injection: parameterized queries used, not string concatenation
- Hardcoded secrets: no credentials, tokens, or keys in code
- Path traversal: file paths validated and sanitized
- Error message leakage: internal errors not exposed to clients

### GO CONVENTIONS
- Interface design: accept interfaces, return structs; interfaces defined at consumer
- Receiver naming: short abbreviation of type name
- Test patterns: table-driven tests, _test.go suffix, TestFunctionName_Scenario naming
- Error wrapping: fmt.Errorf("context: %w", err) pattern
- Package names: lowercase, single word, no underscores

## Output Format

Respond with ONLY a JSON object matching this schema — no markdown fences, no preamble:

{
  "findings": [
    {
      "severity": "bug|warning|info",
      "file": "path/to/file.go",
      "line": 42,
      "message": "Concise description of the issue and why it matters, with a suggested fix."
    }
  ],
  "summary": "Overall assessment: what the change does, key strengths, and the most important issues to address."
}

Severity guide:
- "bug": Incorrect behavior, crashes, data corruption, security vulnerabilities, missing error checks
- "warning": Code smell, performance issue, non-idiomatic Go, maintainability concern
- "info": Style suggestion, minor improvement, informational note

If there are no findings, return an empty findings array with a positive summary. Always return valid JSON.
