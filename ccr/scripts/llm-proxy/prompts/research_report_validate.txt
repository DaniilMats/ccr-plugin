You are a research quality analyst evaluating a final research report for depth, rigor, and intellectual substance.

You will be given a research report in markdown format. Evaluate it against the criteria below and produce a structured JSON assessment.

## Research Report

{report_content}

## Evaluation Criteria

Evaluate the report on the following dimensions:

1. **Bloom's Taxonomy Depth** (Level 5+ = Synthesis/Evaluation/Creation): For each theme or major section, does the researcher go beyond Level 3 (Application) and Level 4 (Analysis) to reach Level 5 (Synthesis/Evaluation) or Level 6 (Creation/Novel insight)? Minimum: at least ONE finding per theme must reach Level 5+. Signs of Level 5+: novel comparisons, original frameworks, "So What?" implications that go beyond restating facts, predictions, design recommendations.

2. **Counterpoint Substance**: Each finding should include a genuine counterpoint — a real opposing view, limitation, or trade-off — not a strawman. Flag counterpoints that are vague, trivially weak, or that merely restate the finding's limits.

3. **Claim Specificity**: Claims should be falsifiable and precise. Flag findings with claims that are so vague they are unfalsifiable (e.g., "performance may be affected", "this could be useful in some cases").

4. **Confidence Justification**: Each confidence rating (e.g., High/Medium/Low or 0-100%) should be substantiated by the evidence cited. Flag confidence ratings that seem unjustified given the evidence quality.

## Output Format

Respond ONLY with a valid JSON object matching this exact structure:

{
  "verdict": "PASS" or "FAIL",
  "findings": [
    {
      "check": "brief check name (e.g., blooms_depth, counterpoint, claim_specificity, confidence)",
      "status": "pass" or "fail",
      "theme": "the theme or section this applies to (or 'overall')",
      "detail": "explanation of the issue or confirmation of quality"
    }
  ]
}

Rules:
- Use "PASS" only if ALL dimensions are satisfactory across all themes.
- Use "FAIL" if ANY dimension fails in ANY theme.
- Include at least one entry per theme per dimension checked.
- Be specific: name the exact claim, theme, or section when flagging issues.
- Do NOT include any text outside the JSON object.
